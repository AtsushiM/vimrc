## 概要

このVim設定では、**fzf.vim**を使用してファイル検索とディレクトリブラウザを実現しています。

### 主な機能

- 🚀 **超高速検索** - ripgrepを使用した高速ファイル検索
- 📁 **ディレクトリブラウザ** - ファイル選択までfzfから抜けない
- 📚 **バッファ＋MRU** - 開いているファイル＋最近使ったファイル
- 🎨 **プレビュー機能** - ファイル内容をその場で確認
- 🔍 **fuzzy検索** - 曖昧検索で素早くマッチ

---

## クイックリファレンス

### 最もよく使うキー

```vim
;uf   " カレントディレクトリ内のファイルを検索
;uu   " バッファ＋最近使ったファイル
;uw   " Worksディレクトリをブラウズ（掘り下げる）
;ua   " Works全体から検索（即座に開く）
```

### ディレクトリブラウザの基本操作

```
;uw または ;ud で起動
  ↓
ディレクトリ選択 → 移動してfzf継続 🔁
  ↓
ファイル選択 → 開いてfzfから抜ける ✅
```

## キーマッピング一覧

### ファイル検索

| キー | 機能 | 詳細 |
|------|------|------|
| **`;uf`** | **カレント検索** | カレントディレクトリ配下を再帰的に検索 |
| **`;ua`** | **Works全体検索** | ~/works/ 配下すべてを検索（即座に開く）|

### ディレクトリブラウザ（掘り下げる）

| キー | 機能 | 詳細 |
|------|------|------|
| **`;uw`** | **Works直下ブラウザ** | ~/works/ 直下をブラウズ（ディレクトリ選択で継続）|
| **`;ud`** | **カレント直下ブラウザ** | カレントディレクトリ直下をブラウズ |

### バッファ・履歴

| キー | 機能 | 詳細 |
|------|------|------|
| **`;uu`** | **バッファ＋MRU** | 開いているバッファ＋最近使ったファイル |
| `;um` | MRUのみ | 最近使ったファイルのみ（バッファ含まず）|
| `;ul` | 行検索 | カレントバッファ内の行を検索 |

### Git・履歴

| キー | 機能 | 詳細 |
|------|------|------|
| `;gf` | Git管理ファイル | Git管理下のファイルのみ検索 |
| `;gs` | Git status | 変更されたファイルのみ |
| `;uc` | コマンド履歴 | コマンド履歴を検索 |
| `;us` | 検索履歴 | 検索履歴を検索 |

### ファイルを開く際のキー

| キー | 動作 |
|------|------|
| `Enter` | 通常で開く |
| `Ctrl-x` | 水平分割で開く |
| `Ctrl-v` | 垂直分割で開く |
| `Ctrl-t` | タブで開く |
| `Ctrl-/` | プレビューのオン/オフ |

## ディレクトリブラウザ

### 基本コンセプト

**ファイルを選択するまでfzfから抜けません**

```
;uw でWorks直下を表示
  ↓
project-a/ を選択 → project-a に移動してfzf継続
  ↓
src/ を選択 → src に移動してfzf継続
  ↓
index.ts を選択 → 開いてfzfから抜ける ✅
```

### 動作の違い

#### `;uw` / `;ud` （ディレクトリブラウザ）

**特徴**:
- ディレクトリ選択 → fzfが再度開く（継続）
- ファイル選択 → ファイルを開いてfzfから抜ける
- `..` で親ディレクトリに戻れる
- 直下のみ表示（情報量が少ない）

**表示例**:
```
Works>
  ..              ← 親ディレクトリに戻る
  project-a/      ← ディレクトリ（選択で移動）
  project-b/      ← ディレクトリ
  README.md       ← ファイル（選択で開く）
```

**用途**: プロジェクトを選んで、ディレクトリを掘り下げながらファイルを探す

#### `;ua` / `;uf` （全体検索）

**特徴**:
- すべてのファイルを再帰的に検索
- ファイル名でfuzzy検索
- 選択したら即座に開く
- ディレクトリ移動はできない

**表示例**:
```
>
  project-a/src/components/Button.tsx
  project-a/src/hooks/useAuth.ts
  project-b/src/utils/helpers.ts
  ...（すべてのファイル）
```

**用途**: ファイル名が分かっていて、素早く開きたい

### 親ディレクトリへの移動

1. `;uw` または `;ud` でブラウザを起動
2. `..` を選択
3. 親ディレクトリに移動してfzfが再度開く

### ディレクトリとファイルの区別

- **ディレクトリ**: `project-a/` のように `/` が付く
- **ファイル**: `README.md` のようにそのまま表示

## 使い分けガイド

### シーン別おすすめキー

#### プロジェクトを選んでファイルを探す

```vim
;uw   " Works直下のプロジェクト一覧（情報量少ない）
      " → project-a/ を選択
      " → src/ を選択
      " → components/ を選択
      " → Button.tsx を選択 → 開く
```

#### ファイル名が分かっている

```vim
;ua   " Works全体を検索
      " → "useAuth" と入力
      " → hooks/useAuth.ts がマッチ
      " → Enter → 即座に開く
```

#### 最近編集したファイルに戻る

```vim
;uu   " バッファ＋MRU
      " → Button.tsx（最近閉じたファイル）
      " → Enter → 開く
```

#### 現在開いているファイルから別のファイルへ

```vim
;ud   " カレント直下をブラウズ
      " → 関連ファイルを探す
      " → Button.test.tsx を選択
```

### キーの使い分け表

| やりたいこと | キー | 理由 |
|------------|------|------|
| プロジェクト選択 | `;uw` | 直下のみで見やすい |
| ディレクトリ構造を把握 | `;ud` | 現在地から掘り下げる |
| ファイル名で検索 | `;ua` | 全体から素早く探せる |
| 最近のファイルに戻る | `;uu` | MRU履歴が便利 |
| カレント内を検索 | `;uf` | 現在のプロジェクト内 |

### `;uw` vs `;ua` の違い

| 項目 | `;uw` | `;ua` |
|------|-------|-------|
| 表示件数 | 少ない（直下のみ）| 多い（すべて）|
| 表示内容 | ディレクトリ中心 | ファイル中心 |
| 動作 | ブラウザ（継続）| 検索（即開く）|
| 用途 | 掘り下げる | 直接開く |

## Tips

### Tip 1: fuzzy検索の活用

完全一致でなくても検索できます：

```
"MyComponent.tsx" を探す場合
→ "mycmp" と入力 → マッチする
→ "mc.tsx" と入力 → マッチする
```

### Tip 2: プレビューのトグル

fzfでファイルを選択中に `Ctrl-/` でプレビューのオン/オフを切り替えられます。

### Tip 3: 複数ファイルを開く

fzfで `Tab` を押すと複数のファイルを選択できます：
1. `Tab` で選択マーク
2. `Shift-Tab` で選択解除
3. `Enter` で複数ファイルを一度に開く

### Tip 4: ディレクトリ階層を素早く移動

```vim
;uw → project-a/ 選択
    → src/ 選択
    → .. 選択（親に戻る）
    → docs/ 選択
    → README.md 選択 → 開く
```

### Tip 5: バッファとMRUの活用

`;uu` を使うと、開いているファイルと最近使ったファイルの両方が表示されるので、素早くファイル間を移動できます。

### Tip 6: Git管理下のファイルのみ検索

```vim
;gf   " Git管理下のファイルのみ
      " → node_modules/ などは除外される
```

### Tip 7: カレントディレクトリを活用

`;ud` でカレントディレクトリ直下をブラウズすると、現在の作業ディレクトリの構造を把握しやすいです。

### Tip 8: MRUで過去のプロジェクトに戻る

別のプロジェクトで作業した後も、`;uu` や `;um` で前のプロジェクトのファイルが履歴に残っているので素早く戻れます。

## よくある質問

### Q: `;uw` でディレクトリを選択してもうまく動かない

**A**: MacVimを完全に再起動してください。設定の変更が反映されていない可能性があります。

```bash
killall MacVim
mvim
```

### Q: MRU（最近使ったファイル）が表示されない

**A**: `;uu` を使ってください。`;uu` はバッファ＋MRUの両方を表示します。MRUのみを表示したい場合は `;um` を使います。

### Q: 親ディレクトリに戻れない

**A**: `;uw` または `;ud` のディレクトリブラウザで `..` を選択してください。全体検索（`;ua`, `;uf`）では親ディレクトリに戻る機能はありません。

### Q: fzf.vimが遅い

**A**: `ripgrep` がインストールされているか確認してください：

```bash
brew install ripgrep
```

### Q: プレビューが表示されない

**A**: `Ctrl-/` を押してプレビューをオンにしてください。または、ウィンドウサイズが小さい場合は表示されないことがあります。

### Q: ディレクトリとファイルの区別がつかない

**A**: ディレクトリには `/` が付きます（例: `project-a/`）。ファイルはそのまま表示されます（例: `README.md`）。
